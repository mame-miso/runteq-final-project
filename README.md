### 本サービスの概要（700文字以内）
「どこに何をしまったか」を写真で管理するアプリです。
収納時に「しまうもの」と「場所」を撮影して、「場所」で紐づけて管理します。

写真で「しまうもの」や「場所」を登録するので、面倒な名前付けや分類の手間がありませんし、
分類や整理が苦手な人でも、しまう場所を登録（写真をとる）だけなので、悩むことなく簡単に利用できます。

主な機能として、保管場所の作成、写真の登録、場所ごとの一覧があります。
これにより、探し物にかかる時間やストレスを軽減し、記憶に頼らない管理ができます。

## 7. このアプリで実現すること
### MVPで作る機能
1)物を保管する場所（例：引き出しAやクローゼット）を写真を撮って登録。
2)その場所に保管するものの写真も撮影し、紐づけて保存。
3)「あれ（物）」を探すときに、アプリを使用してもらい、場所一覧から該当しそうな場所から検索する。

探す人の記憶にある場所に保存されている物の写真が一覧で表示されるため、探している物がそこにあるかどうか、視覚的に見つけることができる。

## 8. このアプリの懸念点とその対策
 写真だけだと分類できない

対策
「場所」をユーザーに作ってもらう（引き出しAなど）
写真はそこに入れるだけ
### 懸念点①低頻度利用でも使い続けてもらえるか
衣替えや収納整理など長期の保存のみに使用された場合、日常的に頻繁に使用されるアプリではない可能性がある。
対策：短期で活用してもらう
・通勤・通学時の持ち物チェック（カバンの中身管理）
・冷蔵庫や日用品ストックの管理に利用
場合によっては日常に欠かせないツールとなる可能性がある。

### 懸念点②：整理が苦手な人に習慣化してもらえるか
対策：「収納するたびに撮影・更新する」習慣をつけてもらうために
・写真を撮るだけで登録できるシンプルな操作にする
・文字入力を極力不要にし、負担を軽減する
・一度登録すれば「探す手間が大きく減る」というメリットを明確に伝える
・チェックリスト機能などを追加し、日常的に使う動機を作る

###懸念点③：登録内容が現実とズレる可能性
物を取り出した後に元の場所へ戻さなかった場合や、アプリ上の情報を更新しなかった場合、実際の保管場所と情報がズレてしまう可能性がある。
対策：完璧な管理を求めず、「目安として使う」前提の設計とする
・よく使う場所のみ登録するなど、負担を減らす
・チェック機能や更新のしやすさを工夫することで、修正のハードルを下げる
