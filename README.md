## 4-4. ユーザーがサービスを導入して利用するまでのイメージ(導入・継続の流れ)
■ ユーザーはなぜ使い続ける？
このアプリは一度登録することで「探し物の時間を減らす」という直接的なメリットがある。
日常生活の中で「どこにしまったか分からない」という場面は繰り返し発生するため、継続的に利用されると考えられる。
また、新しく物を収納するたびに記録する必要があるため、自然と利用が習慣化される。

■ 続けることで何が変わる？
探し物にかかる時間が減少する
収納状況を把握でき、無駄な買い物が減る
長期間使っていない物に気づき、整理・断捨離につながる

■ データが溜まると何ができる？
各収納場所の中身を一覧で把握できる
使用頻度の低い物を可視化できる
「どこに何があるか」を視覚的に確認できるため、記憶に頼らない管理が可能になる

■ 他の人と関わる余地はある？
家族間での利用を想定しており、保管場所や物の情報を共有することで、
「どこに何があるか分からない」という家庭内のコミュニケーションコストを削減できる。
例)
日用品の保管場所共有
防災用品の中身確認
子供の持ち物管理

■ 運用・仕組みで価値を出せないか？
このアプリはシンプルな記録を継続することで価値が高まる設計とする。

例えば：
定期的に内容を確認することで整理整頓を促す
保管場所ごとに情報を整理することで、直感的に探せる状態を維持する

### 10-2. 技術選定の理由
- なぜこの技術を使うのか
Ruby on Railsはこれまでの学習で使用してきたフレームワークであり、
CRUD機能の実装やデータベース連携を効率的に行えるため選定した。
また、本アプリは画像を扱うサービスであるため、
Rails標準のActiveStorageを利用することで、
シンプルな構成で画像アップロード機能を実装できると考えた。
- 今回チャレンジしたい点
ActiveStorageを用いた画像管理機能の実装
モデル同士の関連付け（場所と写真の紐付け）
シンプルで分かりやすいUI設計の実現
- 不安な点
画像アップロード機能の実装や表示処理
データ構造（場所と写真の関係）の設計
